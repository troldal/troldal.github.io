XLCell::XLCell ( XLWorksheet &  parent,
XMLNode  cellNode 
)
protected
Parameters
parentA pointer to the parent XLWorksheet object. Must not be nullptr.
cellNodeA pointer to the XMLNode with the cell data. Must not be nullptr.

This constructor creates a XLCell object based on the cell XMLNode input parameter, and is intended for use when the corresponding cell XMLNode already exist. If a cell XMLNode does not exist (i.e., the cell is empty), use the relevant constructor to create an XLCell from a XLCellReference parameter. The initialization algorithm is as follows:

  1. Is the parameter a nullptr? If yes, throw a invalid_argument exception.
  2. Read the formula and value child nodes. These may be nullptr, if the cell is empty.
  3. Read the address, type and style attributes to the cellNode. The type attribute may be nullptr
  4. set the m_cellReference property using the value of the m_addressAttribute.
  5. Set the cell type, using the previous information:
    1. If there is no type attribute and no value node, the cell is empty.
    2. If there is no type attribute but there is a value node, the cell has a number value.
    3. Otherwise, determine the celltype based on the type attribute.

Consider if this is better done as a free function. What should be the return value?

This methods copies a range into a new location, with the top left cell being located in the target cell. The copying is done in the following way:

  • Define a range with the top left cell being the target cell.
  • Calculate the size of the range from the originating range and set the new range to the same size.
  • Copy the contents of the original range to the new range.
  • Return a reference to the first cell in the new range.
